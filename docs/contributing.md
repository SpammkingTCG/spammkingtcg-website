# Contributing to SpammKing TCG

## Before Making Changes

Read:

- `AGENTS.md`
- `README.md`
- `docs/architecture.md`
- `docs/data-model.md`
- `docs/coding-standards.md`

## Change Philosophy

Every change should answer at least one of these questions:

- Does this help customers buy cards?
- Does this help customers trust SpammKing TCG?
- Does this help suppliers trust SpammKing TCG?
- Does this make the project easier to maintain?

If the answer is no, add it to the backlog instead of building it immediately.

## Workflow

1. Inspect existing files first.
2. Make the smallest useful change.
3. Preserve the public appearance unless the task is explicitly visual.
4. Validate links, JSON and sitemap when relevant.
5. Update docs when structure or workflow changes.
6. Use a clear commit message.

## Commit Messages

Use conventional commit style:

- `feat: add release hub filter`
- `fix: correct set page link`
- `docs: add deployment guide`
- `refactor: simplify product card rendering`
- `style: improve mobile spacing`

## Do Not Add Without Approval

- Direct checkout
- Customer accounts
- Payment integrations
- Major redesigns
- New brand colours
- Large new dependencies
- Public claims about stock, dispatch or supplier relationships that are not confirmed
